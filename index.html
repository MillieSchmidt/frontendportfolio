Life In Pictures (Hebrew)

  • Summary: decided to store my picutres on my own site
  • Primarily for: web, but reponsive
  • Main Technologies: React, JS, Bootstrap
  • Main Challenges: rendering the selected picture

Bookstore (Hebrew)

  • Summary: created a bookstore for the books I authored
  • Primarily for: web, but reponsive
  • Main Technologies: HTML5, CSS3, Bootstrap
  • Main Challenges: creating a functioning navbar in Hebrew to move between all the pages, and a page for my podcast
  • Solutions: took a navbar code from Bootstrap and adjusted it for my needs. As Hebrew is written from right to left, I had to change Bootstrap's structure so it would fit that writing pattern. It was great fun tweaking someone else's code and adjust it to my needs.

Doggo App

  • Summary: choose a dog breed and enjoy the pictures
  • Primarily for: mobile
  • Main Technologies: JavaScript
  • Main Challenges: working with promises
  • Solutions: got an API for watching pictures of dog breeds, from dog.ceo/dog-api/. First step was to fetch the url, then turn it into json, then accessing the information inside the object and retreive whatever is needed. Step by step, one small problem at a time, is the correct way. There were two uses for the data: one was to fill the dropdown menu, and the other was to render the picture of the breed selected by the user. Until the picture is rendered, you see a spinner on screen.

Feed a Mole

  • Summary: a browser game where you need to feed the moles to gain points
  • Primarily for: web
  • Main Technologies: JavaScript
  • Main Challenges: timing the moles
  • Solutions: the moles have several "states": they are either there, or are hungry, or are sad, or are fed, or they are crawling back into their tunnel. Then there's two types of mole: the regular one and the king mole. First I declared all the functions to deal with all those changes, by adding or removing classes. Then I used setInterval and inside it set the setTimeouts to call each function after a certain amount of millieseconds had passed. I know it's less elegant than requestAnimationFrame, but I think for the purpose of this short, simple game, it's good enough.
